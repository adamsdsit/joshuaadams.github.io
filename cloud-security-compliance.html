Cloud Security and Compliance in Practice

Course Code: COM 4XX (Proposed) • Level: Advanced Undergraduate (3 Credits) • Delivery: Hybrid

Course Description

This course introduces students to the principles, technologies, and strategies used to secure modern cloud environments. Emphasis is placed on practical experience with regulatory compliance including HIPAA, FedRAMP, and Japanese data protection regulations such as the Act on the Protection of Personal Information (APPI). Students will also gain hands-on exposure to secure DevOps, incident response, and cloud provider security tools. Individual and team-based projects simulate real-world IT challenges.

Learning Objectives

  • Analyze risks in public, private, and hybrid cloud models
  • Apply regulatory frameworks in secure system design (HIPAA, GDPR, NIST)
  • Implement basic DevSecOps pipelines and access controls
  • Use tools like GCP IAM, Azure Defender, and Terraform for policy enforcement
  • Evaluate cloud infrastructure’s compliance and propose security improvements

Weekly Topics (15 Weeks)

  1. Introduction to Cloud Models and Shared Responsibility
  2. Core Concepts: Confidentiality, Integrity, Availability
  3. Identity & Access Management (IAM)
  4. Network Security and Cloud Perimeters
  5. Compliance Frameworks: HIPAA, FedRAMP, NIST 800-53
  6. Case Study: Cloud Failures and Breaches
  7. Midterm: Threat Modeling and Security Architecture
  8. Logging, Monitoring & SIEM in the Cloud
  9. Infrastructure as Code: Security with Terraform & Ansible
  10. Secure DevOps and CI/CD Pipeline Hardening
  11. Incident Response Planning and Simulation
  12. Business Continuity & Disaster Recovery in the Cloud
  13. Capstone Project Work – Security Audit Report
  14. Project Presentations – Cloud Security Audit
  15. Final Review & Future Trends in Cloud Security
